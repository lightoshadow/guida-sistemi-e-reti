Sezione 2 — Indirizzamento IP

NAT / PAT e Port Forwarding

NAT consente a tutti i dispositivi della rete privata di condividere un singolo IP pubblico per navigare in Internet. Port forwarding espone servizi interni all'esterno.

NAT PAT / NAPT Port forwarding RFC 1918

Perché esiste il NAT

Gli indirizzi IPv4 pubblici sono scarsi (~4 miliardi totali, meno di 1 per persona). Il NAT permette di usare indirizzi privati (RFC 1918) nella rete interna e un singolo IP pubblico verso Internet.

Rete privata Internet 192.168.1.10 ─┐ 192.168.1.11 ─┼─→ [Router NAT] ──→ 203.0.113.1 ──→ Server 192.168.1.12 ─┘ IP pubblico

Tutti i dispositivi privati navigano "come se" avessero lo stesso IP pubblico. Il router tiene traccia delle connessioni per girare le risposte al dispositivo corretto.

Tipi di NAT

Static NAT (NAT statico)

Mappa un IP privato fisso → un IP pubblico fisso. Usato per server che devono essere raggiungibili dall'esterno con IP noto.

192.168.1.10 ↔ 203.0.113.10  (associazione permanente)

Dynamic NAT

Mappa IP privati → IP pubblici da un pool. Non permette connessioni dall'esterno verso l'interno. Raramente usato oggi.

PAT — Port Address Translation (NAT overload)

La forma più comune. Mappa molti IP privati → un solo IP pubblico, differenziandoli tramite la porta sorgente.

IP privato : portaIP pubblico : portaServer di destinazione
192.168.1.10:50100203.0.113.1:501008.8.8.8:443
192.168.1.11:50101203.0.113.1:501018.8.8.8:443
192.168.1.12:50102203.0.113.1:501021.1.1.1:80

Il router mantiene una NAT translation table. Quando arriva la risposta da Internet, cerca la porta e la rimappa all'IP privato corrispondente.

PAT = NAPT = NAT overload. Termini equivalenti. Quasi tutti i router domestici e aziendali usano PAT.

Port Forwarding

Di default, con NAT i dispositivi interni non sono raggiungibili dall'esterno (connessioni entranti vengono bloccate — nessuna entry nella NAT table). Il port forwarding crea una regola statica per redirigere le connessioni entranti verso un host interno.

Configurazione tipica

# Esempio: esporre un web server interno
Regola port forwarding:
  IP esterno (qualsiasi) : porta 443  →  192.168.1.10 : 443
  IP esterno (qualsiasi) : porta 80   →  192.168.1.10 : 80

# Esempio: esporre SSH per gestione remota
  IP esterno (qualsiasi) : porta 2222 →  192.168.1.20 : 22
  (porta esterna diversa per sicurezza)

Sicurezza: il port forwarding espone servizi interni. Limitare sempre gli IP sorgenti ammessi quando possibile. Preferire VPN rispetto al port forwarding diretto per l'accesso remoto alla rete interna.

NAT e IPv6

IPv6 ha abbastanza indirizzi (2¹²⁸) per assegnarne uno pubblico a ogni dispositivo → in teoria NAT non è necessario. Tuttavia, molte reti aziendali usano ancora NAT per privacy e sicurezza.
