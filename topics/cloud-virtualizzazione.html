Sezione 10 — Cloud e Virtualizzazione

Cloud computing e Virtualizzazione

Il cloud e la virtualizzazione sono argomenti frequenti nella seconda parte degli esami. Devi sapere distinguere IaaS/PaaS/SaaS, i tipi di cloud e hypervisor tipo 1 vs tipo 2.

Virtualizzazione — hypervisor tipo 1 e tipo 2

Un hypervisor (o Virtual Machine Monitor) è il software che crea e gestisce le macchine virtuali (VM), virtualizzando CPU, RAM, storage e rete.

Hypervisor tipo 1 (bare-metal)

Gira direttamente sull'hardware fisico, senza OS sottostante. Le VM girano sopra l'hypervisor.

  • Vantaggi: massima performance (accesso diretto hardware), alta densità di VM, usato in produzione
  • Esempi: VMware ESXi/vSphere, Microsoft Hyper-V, KVM (Linux), Citrix Hypervisor

Hypervisor tipo 2 (hosted)

Gira come applicazione sopra un OS host. Le VM sono processi dell'OS host.

  • Vantaggi: facile da installare, ideale per sviluppo/test, nessuna dedicazione hardware
  • Svantaggi: overhead doppio (OS host + hypervisor), performance minore
  • Esempi: Oracle VirtualBox, VMware Workstation/Fusion, QEMU

Container — Docker

I container sono un'alternativa leggera alle VM: isolano un'applicazione e le sue dipendenze senza virtualizzare l'hardware.

VMContainer
IsolamentoHardware virtuale completoProcesso isolato, stesso kernel dell'host
DimensioneGB (include OS completo)MB (solo app + dipendenze)
AvvioMinutiSecondi
OverheadAltoMinimo
Sicurezza isolamentoForte (kernel separato)Minore (stesso kernel)
Uso tipicoAmbienti produzione completi, OS diversiMicroservizi, CI/CD, applicazioni cloud-native

Docker: il motore di container più diffuso. Un'immagine Docker definisce l'applicazione + dipendenze. Il container è un'istanza in esecuzione dell'immagine. Dockerfile → docker build → immagine → docker run → container.
