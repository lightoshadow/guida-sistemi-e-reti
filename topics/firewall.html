Sezione 6 — Sicurezza delle reti

Firewall e Sicurezza Perimetrale

Il firewall compare in 12 esami su 15 — è quasi sempre parte integrante del progetto di rete. Devi sapere i tipi, dove posizionarlo, e come funziona la DMZ.

★ ~12 esami Packet filter Stateful inspection DMZ NGFW

Tipi di firewall — confronto

TipoLivello OSIFunzionamentoPro / Contro
Packet filter L3–L4 Esamina IP sorgente/destinazione, porta, protocollo. Regole statiche ACL. Veloce, semplice. Non vede lo stato della connessione.
Stateful inspection L3–L4 Tiene traccia delle sessioni TCP/UDP in una state table. Permette il traffico di ritorno automaticamente. Più sicuro del packet filter. Non analizza il payload.
Application layer (proxy) L7 Analizza il contenuto applicativo (HTTP, FTP, DNS). Può bloccare contenuti specifici. Molto granulare. Introduce latenza. Deve conoscere ogni protocollo.
NGFW (Next-Gen) L3–L7 Combina stateful + DPI (Deep Packet Inspection) + IDS/IPS + identificazione app + VPN. Soluzione completa. Costo elevato. Esempio: Palo Alto, Fortinet, Cisco ASA.

All'esame: quasi sempre si chiede di scegliere e motivare il tipo di firewall. La risposta migliore è stateful inspection o NGFW, con motivazione sulla visibilità dello stato delle connessioni.

Regole ACL (Access Control List)

Le regole ACL definiscono cosa il firewall permette o blocca. Ogni regola specifica:

  • Sorgente: IP o subnet di partenza
  • Destinazione: IP o subnet di arrivo
  • Protocollo: TCP, UDP, ICMP, Any
  • Porta: numero di porta (es. 80, 443, 22)
  • Azione: ALLOW o DENY

Le regole vengono valutate in ordine. La prima regola che fa match viene applicata. Alla fine c'è sempre una regola implicita "deny all".

# Esempio regole ACL (pseudocodice)
ALLOW  TCP  any          → 10.0.0.10:80     # HTTP verso web server
ALLOW  TCP  any          → 10.0.0.10:443    # HTTPS verso web server
ALLOW  TCP  192.168.1.0/24 → any:22         # SSH dalla rete interna
DENY   ANY  any          → any              # Blocca tutto il resto

DMZ (Demilitarized Zone)

La DMZ è una zona di rete intermedia tra Internet (untrusted) e la rete interna (trusted), dove si collocano i server accessibili dall'esterno (web server, mail server, DNS pubblico).

Architettura a due firewall (consigliata)

Internet │ [FW esterno] ← blocca tutto tranne HTTP/HTTPS verso DMZ │ [DMZ: Web Server, Mail Server, DNS] │ [FW interno] ← blocca tutto tranne traffico legittimo DMZ→LAN │ [Rete interna: PC, Server interni, DB]

Architettura a un firewall con 3 interfacce

Internet ─── eth0 ─── [FIREWALL] ─── eth1 ─── Rete interna (192.168.1.0/24) └──── eth2 ─── DMZ (10.0.0.0/24)

Il firewall gestisce 3 zone: untrust (Internet), DMZ, trust (rete interna). Le regole tipiche:

  • Internet → DMZ: ALLOW solo porte specifiche (80, 443, 25)
  • DMZ → Rete interna: DENY (o solo verso DB specifico)
  • Rete interna → DMZ: ALLOW (per gestione server)
  • Rete interna → Internet: ALLOW

Errore critico da evitare: mettere il database nella DMZ. Il DB deve stare nella rete interna, raggiungibile dal web server della DMZ solo sulla porta specifica (es. 3306 MySQL).

IDS e IPS

IDS (Intrusion Detection System)IPS (Intrusion Prevention System)
ModalitàPassiva (solo rileva)Inline (blocca attivamente)
PosizioneFuori dalla path (port mirror/SPAN)In linea nel traffico
AzioneGenera alert, logGenera alert + blocca traffico
RischioFalsi negativi ignoratiFalsi positivi bloccano traffico legittimo

Metodi di rilevamento: firma (pattern noti di attacchi) vs anomalia (deviazione dal comportamento normale).
