Sezione 5 — Routing e Switching

Routing statico e dinamico

Il routing statico è presente in tutti gli esami: devi saper leggere e compilare tabelle di routing. RIP e OSPF sono richiesti come confronto teorico o configurazione base.

Come funziona il routing

Il router opera a livello 3 (rete). Quando riceve un pacchetto, consulta la sua tabella di routing per decidere dove inviarlo. Confronta l'indirizzo IP di destinazione con le rotte disponibili e sceglie quella con il prefisso più lungo (longest prefix match).

Struttura di una tabella di routing

DestinazioneMascheraNext-hopInterfacciaMetrica
192.168.1.0/24directly connectedeth00
192.168.2.0/2410.0.0.2eth11
10.0.0.0/30directly connectedeth10
0.0.0.0/0203.0.113.1eth21

La riga 0.0.0.0/0 è il default gateway (rotta di default): usata per tutti i pacchetti che non corrispondono a nessuna rotta più specifica.

Longest prefix match: tra più rotte che corrispondono, vince quella con la maschera più lunga (più specifica). Es.: se esistono /24 e /16, la /24 batte sempre la /16.

Routing statico

Le rotte sono configurate manualmente dall'amministratore. Non si aggiornano automaticamente in caso di guasti.

# Cisco IOS — aggiungere una rotta statica
ip route 192.168.2.0 255.255.255.0 10.0.0.2
#         ^destinazione  ^maschera  ^next-hop

# Rotta di default
ip route 0.0.0.0 0.0.0.0 203.0.113.1
  • Pro: semplice, predicibile, nessun overhead di protocollo, sicuro (nessun routing protocol da attaccare)
  • Contro: non scala (in reti grandi serve configurare ogni rotta manualmente), nessun failover automatico
  • Quando usarlo: reti piccole (<5 router), link WAN verso ISP, stub network (rete con un solo percorso)

Routing dinamico — RIP

Il RIP (Routing Information Protocol) è un protocollo distance-vector. Ogni router comunica la propria tabella di routing ai vicini diretti ogni 30 secondi.

  • Metrica: hop count (numero di router attraversati)
  • Limite: max 15 hop (16 = rete irraggiungibile)
  • Convergenza: lenta (può richiedere minuti dopo un guasto)
  • Versioni: RIPv1 (classful, broadcast) → RIPv2 (classless, multicast 224.0.0.9, supporta VLSM)

Limitazione critica: il limite di 15 hop rende RIP inadatto a reti grandi. Oggi è usato solo in ambienti molto piccoli o per scopi didattici.

Routing dinamico — OSPF

Il OSPF (Open Shortest Path First) è un protocollo link-state. Ogni router mappa l'intera topologia della rete e calcola il percorso ottimale con l'algoritmo di Dijkstra (SPF — Shortest Path First).

  • Metrica: costo (basato sulla banda: costo = 10⁸/bandwidth)
  • Nessun limite di hop
  • Convergenza rapida (pochi secondi dopo un guasto)
  • Gerarchico: suddivide la rete in aree (Area 0 = backbone)
  • Multicast: 224.0.0.5 (tutti i router OSPF), 224.0.0.6 (DR/BDR)

Inter-VLAN routing (router-on-a-stick)

Per instradare il traffico tra VLAN diverse serve un router (o un L3 switch). Con router-on-a-stick:

  1. Il link router↔switch è configurato come trunk
  2. Il router ha una sub-interfaccia per ogni VLAN, ognuna con il proprio IP gateway
  3. Il traffico inter-VLAN sale sul trunk, viene instradato, scende verso la VLAN di destinazione
# Configurazione switch (lato trunk)
interface GigabitEthernet0/24
  switchport mode trunk
  switchport trunk allowed vlan 10,20,30

# Configurazione router (sub-interfacce)
interface GigabitEthernet0/0.10
  encapsulation dot1Q 10
  ip address 192.168.10.1 255.255.255.0

interface GigabitEthernet0/0.20
  encapsulation dot1Q 20
  ip address 192.168.20.1 255.255.255.0
