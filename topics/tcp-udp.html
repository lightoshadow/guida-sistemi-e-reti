Sezione 7 — Protocolli di trasporto

TCP vs UDP — livello di trasporto

TCP e UDP sono i due protocolli fondamentali del livello 4 (trasporto). Ogni servizio di rete usa uno dei due — devi sapere quale e perché.

SYN flood attack: un attaccante invia molti SYN senza completare l'handshake, esaurendo le risorse del server. Contromisura: SYN cookies.

Numero di sequenza e ACK

TCP numera ogni byte trasmesso. Il ricevitore invia ACK per ogni segmento ricevuto, specificando il numero del prossimo byte atteso. Se un segmento non viene confermato entro il timeout, viene ritrasmesso.

Porte TCP/UDP

Le porte (0–65535) identificano il servizio/applicazione su un host:

  • Well-known ports (0–1023): assegnate a servizi standard (HTTP 80, HTTPS 443, SSH 22…)
  • Registered ports (1024–49151): registrate per applicazioni specifiche
  • Dynamic/ephemeral ports (49152–65535): assegnate dal SO ai client per le connessioni in uscita

Una connessione TCP è identificata da 4 valori (socket): IP sorgente : porta sorgente → IP destinazione : porta destinazione

ICMP — Internet Control Message Protocol

ICMP opera a livello 3 (rete), non è un protocollo di trasporto ma di diagnostica. Non usa porte — usa type e code.

Comandi diagnostici

ComandoMessaggi ICMP usatiFunzione
pingEcho Request (type 8) / Echo Reply (type 0)Verifica raggiungibilità di un host
tracerouteTime Exceeded (type 11)Mostra il percorso hop-by-hop

Come funziona traceroute

  1. Invia pacchetti con TTL = 1. Il primo router decrementa TTL a 0 → scarta e risponde con "Time Exceeded"
  2. Invia con TTL = 2 → il secondo router risponde
  3. Continua aumentando TTL finché non si raggiunge la destinazione

Ogni hop risponde con il suo IP → si costruisce il percorso completo.

All'esame (troubleshooting): ping per testare connettività, traceroute per localizzare dove si interrompe il percorso.
