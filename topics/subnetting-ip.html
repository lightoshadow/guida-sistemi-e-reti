Sezione 2 — Indirizzamento IP

Indirizzamento IP e Subnetting

Il calcolo degli indirizzi e la divisione in sottoreti è presente in ogni singola traccia d'esame. Devi saper eseguire tutto a mano, senza calcolatore.

Indirizzi IPv4 — struttura

Un indirizzo IPv4 è composto da 32 bit, scritti come 4 ottetti decimali separati da punti (notazione dotted-decimal). Ogni ottetto vale 0–255.

L'indirizzo si divide in due parti: parte di rete (identificata dalla subnet mask) e parte host.

Classi storiche e indirizzi privati (RFC 1918)

ClasseRangeIndirizzi privatiMask default
A1.0.0.0 – 126.255.255.25510.0.0.0/8/8
B128.0.0.0 – 191.255.255.255172.16.0.0/12/16
C192.0.0.0 – 223.255.255.255192.168.0.0/16/24

Attenzione: 127.0.0.0/8 è riservato al loopback (localhost). 169.254.0.0/16 è APIPA (assegnato automaticamente senza DHCP).

Notazione CIDR e subnet mask

La notazione CIDR (Classless Inter-Domain Routing) scrive la maschera come /n dove n è il numero di bit a 1 nella subnet mask.

Formule fondamentali

N° host utilizzabili = 2^(32−n) − 2 N° sottoreti = 2^(bit presi dagli host per le subnet)

Si sottraggono 2 perché l'indirizzo di rete (tutti bit host = 0) e di broadcast (tutti bit host = 1) non sono assegnabili.

Come calcolare da CIDR → componenti della subnet

Dato 192.168.10.0/26:

  1. Subnet mask: /26 → 255.255.255.192 (gli ultimi 6 bit sono host = 64+128=192)
  2. Indirizzo di rete: 192.168.10.0
  3. Indirizzo di broadcast: 192.168.10.63 (0 + 64 − 1 = 63)
  4. Range host: 192.168.10.1 – 192.168.10.62
  5. N° host: 2⁶ − 2 = 62

Trucco: il "salto" tra subnet consecutive = 2^(bit host). Con /26 il salto è 64. Le subnet di 192.168.10.0/24 divisa in /26 sono: .0, .64, .128, .192.

Subnetting — procedura passo-passo

Esercizio tipico d'esame: "Dividere 10.0.0.0/8 per ospitare le seguenti sottoreti: Rete A (200 host), Rete B (100 host), Rete C (50 host), Rete D (25 host), link WAN (2 host)."

Passo 1 — Scegliere la maschera minima

Per ogni rete, trovare il prefisso più corto tale che 2^(32−n) − 2 ≥ host_richiesti:

ReteHost richiesti2^x ≥ richiesti+2PrefissoHost usabili
A2002⁸=256/24254
B1002⁷=128/25126
C502⁶=64/2662
D252⁵=32/2730
WAN22²=4/302

Passo 2 — Assegnare gli spazi (con VLSM, ordine decrescente)

Con VLSM si assegnano prima le subnet più grandi per non sprecare spazio:

10.0.0.0/24   → Rete A  (network: 10.0.0.0, bcast: 10.0.0.255)
10.0.1.0/25   → Rete B  (network: 10.0.1.0, bcast: 10.0.1.127)
10.0.1.128/26 → Rete C  (network: 10.0.1.128, bcast: 10.0.1.191)
10.0.1.192/27 → Rete D  (network: 10.0.1.192, bcast: 10.0.1.223)
10.0.1.224/30 → WAN     (network: 10.0.1.224, bcast: 10.0.1.227)

VLSM (Variable Length Subnet Masking) = subnet con maschere di lunghezza variabile nello stesso spazio di indirizzi. Permette di ottimizzare l'uso degli indirizzi.

Il gateway è sempre il primo host utilizzabile della subnet (convenzione standard, anche se tecnicamente il gateway può essere qualsiasi host della rete).

Indirizzi statici vs DHCP

  • Statici: server, stampanti di rete, gateway, access point — dispositivi che devono sempre rispondere allo stesso IP.
  • DHCP dinamico: PC, notebook, smartphone, qualsiasi client che può ricevere un IP temporaneo (lease).
  • Riserva un range per DHCP ed escudi gli statici: es. host .1–.10 statici, DHCP scope .11–.254.

Come compare all'esame

  • Prima parte (progetto): ti viene dato un edificio con N reparti; devi dimensionare le subnet e compilare il piano di indirizzamento.
  • Seconda parte (quesiti): domande su CIDR, calcolo di broadcast/host, VLSM.
  • Spesso richiedono di mostrare le subnet in forma tabellare con almeno 5 colonne (rete, mask, gateway, range, broadcast).
  • Frequente: partizionare un /16 o /24 in sottoreti di dimensioni diverse con VLSM.

Errori frequenti: dimenticare di sottrarre 2 agli host, confondere l'indirizzo di rete con il gateway, non ordinare le subnet da grande a piccola con VLSM.
