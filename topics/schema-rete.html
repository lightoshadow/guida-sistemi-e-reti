Sezione 1 — Architettura di rete

Progettazione della rete e schema logico/fisico

La progettazione dell'infrastruttura è il cuore della prima parte dell'esame. Devi saper descrivere, disegnare e motivare l'architettura scelta, includendo tutti i dispositivi e le loro interconnessioni.

Schema logico vs schema fisico

Schema logicoSchema fisico
MostraSubnet, VLAN, indirizzi IP, flussi di traffico, zone di sicurezzaDispositivi fisici, cavi, armadi rack, posizione negli edifici
IncludeRouter, switch (logici), firewall, server, AP — con IP e subnetPatch panel, percorsi cavi, centri stella, prese RJ45
ScalaNon in scala — mostra la topologiaPuò rispettare la planimetria dell'edificio
All'esameQuasi sempre richiesto — disegna le subnet con IPSpesso richiesto insieme allo logico

Consiglio: inizia sempre dallo schema logico. Prima decidi le VLAN e le subnet, poi il fisico ne è la conseguenza.

Componenti da inserire nello schema

Dispositivi obbligatori in quasi ogni progetto

  • Router/firewall di bordo: interfaccia verso l'ISP/Internet — con IP pubblico e privato (gateway)
  • Switch core (L3): centro stella, connette tutti i distribution switch e server
  • Distribution switch: uno per edificio/piano, connesso al core via fibra
  • Access switch: uno per sala/reparto, connesso al distribution
  • Access point: per la copertura wireless, connesso all'access switch
  • Server: web server (in DMZ), file server, DHCP/DNS, mail server

Dispositivi condizionali

  • Firewall separato: se si chiede DMZ
  • VPN concentrator: se ci sono connessioni remote
  • IDS/IPS: se richiesto dalla traccia
  • Load balancer: per server ad alta disponibilità

Esempio schema logico tipo

Come motivare le scelte progettuali

All'esame non basta disegnare — devi motivare ogni scelta. Struttura della risposta:

  1. Identificare i requisiti dalla traccia (quanti utenti, quali servizi, sicurezza richiesta)
  2. Segmentare la rete in VLAN coerenti con i reparti/funzioni
  3. Scegliere le subnet dimensionate ai requisiti (VLSM)
  4. Posizionare i dispositivi di sicurezza (firewall, DMZ, IDS)
  5. Pianificare la ridondanza se richiesta (doppi link, STP, clustering)

Frasi utili per motivare

  • "La rete è segmentata in VLAN per limitare il broadcast domain e aumentare la sicurezza"
  • "I server pubblici sono collocati in DMZ per separarli dalla rete interna"
  • "Il firewall stateful inspection filtra il traffico tra Internet, DMZ e rete interna"
  • "La VLAN ospiti è isolata dalla rete aziendale per prevenire accessi non autorizzati alle risorse interne"
  • "Il cablaggio strutturato su fibra ottica garantisce le prestazioni necessarie tra i piani"

Topologie di rete

TopologiaSchemaVantaggiSvantaggi
StellaTutti i nodi connessi a un hub/switch centraleFacile gestione, guasto di un nodo non blocca gli altriIl dispositivo centrale è un single point of failure
BusTutti i nodi su un unico cavo condivisoSemplice, economicoUn guasto blocca tutta la rete, obsoleto
AnelloNodi collegati in cerchioNessuna collisione, determinismoUn guasto può bloccare l'anello (senza ridondanza)
Albero (gerarchica)Stella di stelleScalabile, standard nelle reti aziendaliIl core è critico
Maglia (mesh)Ogni nodo connesso a più nodiAlta ridondanzaCostoso, complesso

Nelle reti aziendali moderne si usa la topologia gerarchica a tre livelli: Core (backbone) → Distribution (aggregazione) → Access (endpoint). È la topologia che compare in quasi tutti i progetti d'esame.
