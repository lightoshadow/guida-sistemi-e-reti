Sezione 7 — Protocolli applicativi

DHCP — Sequenza DORA

Il DHCP assegna automaticamente gli indirizzi IP ai dispositivi della rete. La sequenza DORA (Discover → Offer → Request → Acknowledge) è un classico all'esame.

UDP 67/68 DORA Lease Relay agent

Sequenza DORA

Quando un dispositivo si collega alla rete e non ha ancora un indirizzo IP, avvia la procedura DORA (comunicazione in broadcast perché non conosce ancora il server):

D
Discover — DHCP Discover Il client invia in broadcast (255.255.255.255) sulla porta UDP 67. "Qualcuno può darmi un indirizzo IP?" — usa MAC address come identificatore.
O
Offer — DHCP Offer Il server risponde (ancora in broadcast) con una proposta: "Ti offro l'IP 192.168.1.50 per 24 ore, il tuo gateway è 192.168.1.1, DNS 8.8.8.8".
R
Request — DHCP Request Il client accetta l'offerta e lo comunica in broadcast (altri server DHCP vedono che la loro offerta è stata rifiutata). "Accetto l'offerta del server X."
A
Acknowledge — DHCP Acknowledge Il server conferma l'assegnazione e imposta il lease time. Il client può ora usare l'indirizzo.

Porte: il server DHCP ascolta su UDP 67. Il client usa la porta UDP 68. I messaggi broadcast usano indirizzo sorgente 0.0.0.0 (il client non ha ancora IP).

Parametri assegnati dal DHCP

Il DHCP non assegna solo l'indirizzo IP. Con ogni lease vengono forniti:

Opzione DHCPN°Descrizione
IP Address—L'indirizzo IP assegnato al client
Subnet Mask1La maschera della rete (es. 255.255.255.0)
Default Gateway3Il router della rete locale
DNS Server6Uno o più server DNS
Lease Time51Durata del lease (es. 86400 secondi = 24 ore)
Domain Name15Suffisso DNS del dominio locale
NTP Server42Server di sincronizzazione orario

Scope, esclusioni e riserve

Scope

Il pool di indirizzi che il server DHCP può assegnare. Esempio:

Scope: 192.168.1.0/24
Range assegnabile: 192.168.1.10 – 192.168.1.200
Gateway: 192.168.1.1
DNS: 192.168.1.2, 8.8.8.8
Lease time: 24 ore

Esclusioni (exclusions)

Indirizzi nel range dello scope che NON devono essere assegnati (perché usati per dispositivi statici):

Esclusioni: 192.168.1.1 – 192.168.1.9  (gateway, server, stampanti)

Riserve (reservations)

Associazione MAC address → IP fisso: il server assegna sempre lo stesso IP a uno specifico dispositivo (in base al MAC). Utile per stampanti e server che necessitano di IP prevedibile ma gestito dal DHCP.

Best practice: assegna staticamente solo i pochi dispositivi critici (router, server). Per stampanti e AP usa le riserve DHCP — più semplice da gestire centralmente.

DHCP Relay Agent

Il DHCP usa messaggi broadcast, ma i broadcast non attraversano i router. Nelle reti con più subnet, ogni subnet dovrebbe avere un server DHCP — soluzione poco scalabile.

Il DHCP Relay Agent (configurato sul router) ascolta i broadcast DHCP locali e li inoltra in unicast al server DHCP centralizzato in un'altra subnet.

Configurazione del relay su un router Cisco:

interface Vlan10
  ip helper-address 192.168.1.100   ← indirizzo del server DHCP

All'esame: se si chiede come un singolo server DHCP serve più VLAN, la risposta è DHCP Relay Agent (ip helper-address).
