Sezione 12 — Programmazione di rete

Socket e programmazione di rete

I socket sono l'astrazione che permette alle applicazioni di comunicare via rete. Compare negli esami con domande su sequenze di chiamate, struttura client-server e protocolli applicativi personalizzati.

Socket TCP Socket UDP Client-server Python

Cos'è un socket

Un socket è un endpoint di comunicazione identificato da:

  • Indirizzo IP: identifica l'host
  • Porta: identifica il processo/servizio sull'host
  • Protocollo: TCP o UDP

Una connessione TCP è identificata dalla coppia di socket: (IP_src:porta_src, IP_dst:porta_dst). Questo permette allo stesso server di gestire migliaia di connessioni sulla stessa porta (es. 443).

Sequenza socket TCP — client/server

SERVER

import socket

# 1. Crea socket
s = socket.socket(
  socket.AF_INET,      # IPv4
  socket.SOCK_STREAM   # TCP
)

# 2. Associa porta
s.bind(("0.0.0.0", 8080))

# 3. Ascolta connessioni
s.listen(5)  # backlog=5

while True:
  # 4. Accetta connessione
  conn, addr = s.accept()

  # 5. Riceve/invia dati
  data = conn.recv(1024)
  conn.send(b"risposta")

  conn.close()

CLIENT

import socket

# 1. Crea socket
s = socket.socket(
  socket.AF_INET,
  socket.SOCK_STREAM
)

# 2. Connette al server
s.connect(("192.168.1.1", 8080))

# 3. Invia dati
s.send(b"richiesta")

# 4. Riceve risposta
data = s.recv(1024)
print(data)

s.close()

Mnemonico sequenza server: socket() → bind() → listen() → accept() → recv/send() → close()

Socket UDP

UDP non ha connessione — non serve connect(), listen(), accept(). Si usa sendto() e recvfrom() che includono l'indirizzo.

# Server UDP
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
s.bind(("0.0.0.0", 5005))
data, addr = s.recvfrom(1024)   # ritorna anche l'indirizzo mittente
s.sendto(b"risposta", addr)

# Client UDP
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
s.sendto(b"dati", ("192.168.1.1", 5005))
data, _ = s.recvfrom(1024)

Protocollo applicativo personalizzato

All'esame spesso si chiede di definire il formato dei messaggi scambiati tra client e server (protocollo applicativo). Opzioni:

Formato testuale (JSON)

# Messaggio client → server
{"tipo": "LETTURA", "sensore": "temperatura", "id": 1}

# Risposta server → client
{"stato": "OK", "valore": 22.5, "unita": "°C", "timestamp": 1715000000}

Formato testuale semplice (CSV/campi fissi)

# Struttura: COMANDO;PARAMETRO1;PARAMETRO2\n
GET;temperatura;sala\n
OK;22.5;°C\n

Considerazioni

  • Delimitatore: come si sa quando finisce un messaggio? (newline \n, lunghezza fissa, lunghezza nel campo header)
  • Encoding: UTF-8 per testo, big-endian per numeri binari
  • Checksum: per verificare l'integrità dei dati (CRC, MD5)

Server multi-client con thread

Un server che gestisce un client alla volta è bloccante. Per servire più client contemporaneamente si usa un thread per connessione:

import socket, threading

def gestisci_client(conn, addr):
    print(f"Connesso: {addr}")
    while True:
        data = conn.recv(1024)
        if not data:
            break
        conn.send(data.upper())
    conn.close()

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(("0.0.0.0", 8080))
server.listen(10)

while True:
    conn, addr = server.accept()
    t = threading.Thread(target=gestisci_client, args=(conn, addr))
    t.daemon = True
    t.start()

Race condition: se più thread accedono a una risorsa condivisa (es. lista di clienti connessi) senza sincronizzazione → errori imprevedibili. Soluzione: threading.Lock() (mutex).
