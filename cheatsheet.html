Formule — Link Budget e Fibra
Free-Space Path Loss (FSPL) FSPL (dB) = 20·log₁₀(d) + 20·log₁₀(f) + 32.4
d in km, f in MHz
Potenza ricevuta P_RX = P_TX + G_TX + G_RX − FSPL − altre perdite
tutti i valori in dBm / dBi / dB
Margine di collegamento Margine = P_RX − Sensibilità_RX
deve essere > 0 dB (tipicamente ≥ 10 dB)
Attenuazione fibra SMF (monomodale) ~0.2 dB/km @ 1550 nm
usata per lunghe distanze (> km)
Attenuazione fibra MMF (multimodale) ~3.5 dB/km @ 850 nm
usata per LAN / edificio (centinaia di m)

Tipi di firewall
Tipo Livello OSI Caratteristiche
Packet Filter L3 / L4 Regole su IP src/dst, porta, protocollo. Stateless. Veloce ma limitato.
Stateful Inspection L4 Tiene traccia delle connessioni (tabella di stato). Blocca pacchetti fuori sessione.
Application-layer (NGFW) L7 Deep packet inspection, IDS/IPS integrato, controllo utente, SSL inspection.
Proxy firewall L7 Intermediario per ogni connessione. Nasconde topologia interna. Circuit/application proxy.
WAF L7 Specializzato per HTTP/S. Protegge da SQLi, XSS, CSRF. Davanti a web server.

Tipo Algoritmi Chiavi Uso principale
Simmetrica AES-128/256, 3DES, ChaCha20 Unica (stessa per cifrare e decifrare) Cifratura dati in transito/a riposo, VPN payload

VLAN 802.1Q — trunk e access
Concetto Dettaglio
Porta access Un'unica VLAN; il frame esce senza tag 802.1Q. Usata per PC, stampanti, IP phone.
Porta trunk Più VLAN; i frame trasportano il tag 802.1Q (4 byte, campo VID 12 bit → max 4094 VLAN). Usata tra switch e verso router.
VLAN nativa VLAN il cui traffico viaggia sul trunk senza tag (default VLAN 1). Cambiare per sicurezza.
Inter-VLAN routing Router-on-a-stick: sub-interfacce sul router (int Fa0/0.10, encapsulation dot1Q 10). Oppure L3 switch con SVI.
Vantaggi VLAN Segmentazione broadcast, sicurezza, flessibilità topologica, QoS separata (es. VLAN VoIP, VLAN IoT, VLAN guest).
Tag 802.1Q (4 byte inseriti nel frame Ethernet) TPID (0x8100) | PCP (3 bit) | DEI (1 bit) | VID (12 bit)

Routing statico e dinamico (RIP / OSPF)
Tipo Caratteristiche chiave Uso
Statico Configurazione manuale (ip route rete mask next-hop). Default route: 0.0.0.0/0. Nessun overhead, pieno controllo. Reti piccole, default GW
RIP v2 Distance-vector. Metrica: hop count (max 15). Aggiornamento broadcast ogni 30 s. Convergenza lenta. Reti semplici / laboratorio
OSPF Link-state. Algoritmo Dijkstra (SPF). Metrica: costo (100 Mbps / banda). Aree (Area 0 = backbone). Convergenza rapida. Reti medie-grandi aziendali
Cisco — route statica ip route 192.168.2.0 255.255.255.0 10.0.0.1
rete destinazione mask next-hop (o interfaccia uscita)
Default route (gateway of last resort) ip route 0.0.0.0 0.0.0.0 <next-hop>

DMZ — De-Militarized Zone
Zona Accesso tipico Contiene
Internet Untrusted Utenti esterni, indirizzi pubblici
DMZ Semi-trusted Web server, mail server, DNS pubblico, reverse proxy
LAN interna Trusted Workstation, server interni, AD, database
Regola firewall tipica (dual-firewall o singolo a 3 interfacce) Internet → DMZ: PERMIT tcp dst 80,443 (web pubblico)
Internet → LAN: DENY any (nessun accesso diretto)
DMZ → LAN: DENY any (server DMZ non vedono LAN)
LAN → DMZ: PERMIT (amministrazione)
LAN → Internet: PERMIT (navigazione)

VPN site-to-site — IPsec
Parametro Tunnel mode Transport mode
Cosa cifra Intero pacchetto IP originale (+ nuovo header IP) Solo il payload (header IP originale visibile)
Uso tipico VPN gateway-to-gateway (site-to-site) Host-to-host (es. IPsec nativo tra server)
AH Integrità + autenticazione; NON cifra. Incompatibile con NAT.
ESP Cifratura (AES) + integrità + autenticazione. Usato in pratica.
Fasi IKE (negoziazione) Fase 1: autenticazione peer + SA ISAKMP (canale sicuro)
Fase 2: negoziazione SA IPsec (parametri ESP/AH, chiavi di sessione)
Tipi di VPN a confronto Site-to-site: gateway ↔ gateway, permanente, trasparente
Client-to-site: laptop/smartphone ↔ gateway, su richiesta
SSL/TLS VPN: porta 443, attraversa NAT/firewall facilmente

Proxy server · Business Continuity · GDPR
Proxy server
Forward proxyClient → Proxy → Internet. Filtra contenuti, fa caching, nasconde IP client. Usato in azienda per controllo navigazione.
Reverse proxyInternet → Proxy → Server interni. Bilancia il carico, termina TLS, protegge backend. Es. Nginx, HAProxy.
NAT vs ProxyNAT opera a L3 (IP), Proxy a L7 (applicazione). Il proxy può ispezionare il contenuto HTTP.
Transparent proxyIl client non sa di passare dal proxy (configurazione sul router/firewall).
Business Continuity
RTORecovery Time Objective — tempo massimo tollerabile di inattività dopo il guasto.
RPORecovery Point Objective — quantità massima di dati persi (misurata nel tempo).
BackupFull: copia totale (lento, completo). Incrementale: solo modifiche dall'ultimo backup. Differenziale: modifiche dall'ultimo full.
RAIDRAID 1: mirror (ridondanza). RAID 5: parità distribuita (min 3 dischi). RAID 10: mirror+stripe.
Cluster HAFailover automatico, load balancing, hot standby per eliminare SPOF.
GDPR (Reg. UE 2016/679)
TitolareDecide finalità e mezzi del trattamento. Responsabile delle misure di sicurezza.
ResponsabileTratta i dati per conto del titolare (es. fornitore cloud). Contratto DPA obbligatorio.
DPOData Protection Officer — obbligatorio per PA, grandi aziende, trattamenti su larga scala.
DirittiAccesso, rettifica, cancellazione ("diritto all'oblio"), portabilità, opposizione, limitazione.
Notifica breachAll'autorità (Garante) entro 72 ore. All'interessato se rischio elevato.
SanzioniFino a 20 M€ o 4% fatturato globale annuo (il maggiore).
